The Foundation

AOS Starts Where EOS Left Off

The Entrepreneurial Operating System (EOS) gave growing companies a shared language and operating rhythm. AOS keeps every piece that works ‚Äî and adds a layer EOS never had: the ability to turn repeating work into permanent infrastructure.

üéØ

Vision/Traction Organizer (V/TO)

The strategic document. 10-year target, 3-year picture, 1-year plan, quarterly Rocks. Gets the whole team on the same page about where you're going.

ü™®

Rocks

The 3-7 most important things each quarter. Not a to-do list ‚Äî a commitment. Every person has Rocks. Every Rock has an owner. Every quarter, you score them.

üìä

Scorecard

5-15 numbers the leadership team reviews weekly. Leading indicators, not lagging. You see problems before they become fires.

üìã

Issues List

A living list of problems, ideas, and obstacles. Anything that's unresolved goes here. You process it with IDS: Identify, Discuss, Solve.

üîü

Level 10 Meeting (L10)

The weekly pulse. Same day, same time, same agenda. Scorecard review, Rock check-in, headlines, and most of the time spent solving issues using IDS.

üí°

IDS (Identify, Discuss, Solve)

The decision-making process. Don't talk around problems ‚Äî name it, debate it, decide on the action. Then it becomes a To-Do with an owner and a deadline.

EOS Stops Here

  • Issue surfaces in the L10 meeting
  • Team uses IDS to solve it
  • It becomes a To-Do (assigned to a person)
  • Person does it. Checks it off next week.
  • If it recurs‚Ä¶ the team solves it again. And again.
‚Üí

AOS Continues

  • Issue surfaces the same way ‚Äî meetings, feedback, data
  • Team solves it the first time (Task)
  • If it recurs, the team documents it (SOP)
  • If it can be triggered by a human, build a Skill
  • If it should run without anyone watching, build an Agent
  • The issue never needs solving again.
AOS doesn't replace EOS. It extends the loop. EOS gives you the rhythm ‚Äî the meetings, the Rocks, the Scorecard. AOS gives you the upgrade path ‚Äî so the things you solve this quarter become infrastructure you never touch again.

Why AOS Matters for This Ecosystem

The RUFORUM and Summit work has a problem EOS never had to solve: the units aren't all under one roof. You're coordinating across 17+ universities, multiple governments, private sector partners, and investor networks spread across continents. That makes the Agent layer disproportionately important. In a single-office business, you can walk over and ask someone. In a distributed ecosystem, the things that fall through cracks are the things nobody's watching between meetings.

If one university needs it, it's a task.
If every university needs it, it's an SOP.
If it needs to run across all nodes without someone chasing each one, it's an agent.

The AOS Quarterly

Same structure as an EOS quarterly. You score last quarter's Rocks, set new ones, and review the V/TO. AOS adds one new artifact: the Infrastructure Ledger.

Standard Quarterly Review

  • Score last quarter's Rocks (complete/incomplete)
  • Review the V/TO ‚Äî is the vision still right?
  • Set next quarter's Rocks (3-7 per person)
  • Clear the Issues List ‚Äî what's been lingering?
  • Revisit the Scorecard ‚Äî right numbers?
+

AOS Infrastructure Ledger

  • How many issues got promoted to SOPs this quarter?
  • How many SOPs became Skills or Agents?
  • What's still stuck as a recurring task someone does manually?
  • Which agents failed? What did we learn?
  • What % of our operating load runs on infrastructure vs. human effort?
The goal each quarter: more of the team's operating load handled by infrastructure, freeing up human attention for the problems that actually need judgment. Not more hours. More leverage.
